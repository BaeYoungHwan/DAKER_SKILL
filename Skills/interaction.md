# Skills — 상태 관리 및 UI 흐름 규칙

## 1. Session State 초기화 규칙

앱 최초 로드 시 사이드바 진입 전 4개의 상태 키를 초기화한다.

| 키 | 초기값 | 초기화 조건 |
|----|--------|-------------|
| `selected_tickers` | `watchlist.json` 내용 (없으면 `["AAPL", "MSFT", "GOOGL"]`) | 키 미존재 시 1회 |
| `ticker_names` | `{}` | 키 미존재 시 1회 |
| `search_counter` | `0` | 키 미존재 시 1회 |
| `holdings` | `{}` | 키 미존재 시 1회 |

---

## 2. 워치리스트 영속성 규칙

종목 목록은 `watchlist.json`(프로젝트 루트)에 UTF-8 JSON 배열로 저장된다.

- **로드 시점**: 앱 최초 로드 시 `selected_tickers` 초기화 단계
- **저장 시점**: 종목 추가, 종목 제거 직후 즉시 저장
- **저장 포맷**: `["005930.KS", "AAPL", ...]` (순서 유지)
- **오류 처리**: 읽기/쓰기 실패 시 예외를 무시하고 기본값 사용 (앱 중단 방지)

---

## 3. 검색 카운터 패턴

Streamlit은 동일 `key`의 입력 위젯을 리렌더 시 값을 유지한다. 종목 추가 후 검색창을 자동 초기화하기 위해 카운터 기반 키 교체 패턴을 사용한다.

```
1. 종목 추가 완료
2. search_counter += 1
3. st.rerun() 호출
4. 재렌더 시 key = f"search_q_{search_counter}" → 새 키로 빈 입력창 생성
```

- `search_counter` 값은 앱 생애주기 동안만 유지 (재시작 시 0으로 초기화됨)

---

## 4. URL 파라미터 규칙

외부 링크로 초기 종목을 전달할 때 URL 쿼리 파라미터를 사용한다.

- **형식**: `?tickers=005930.KS,000660.KS,AAPL`
- **파싱**: 쉼표 구분, 각 값 strip + uppercase 처리
- **중복 방지**: `_params_loaded` 플래그로 페이지 내 재로드 시 중복 추가 방지
- **적용 조건**: `selected_tickers` 초기화 이후에 파라미터를 병합 (초기값 덮어쓰지 않음)

---

## 5. Holdings 관리 규칙

포트폴리오 손익 계산을 위해 매입 정보를 세션 상태에 임시 보관한다.

- **키 구조**: `{ticker: {"avg_cost": float, "quantity": float}}`
- **avg_cost**: 평균 매입 단가 (원화 또는 달러, 종목 통화 기준)
- **quantity**: 보유 수량 (주)
- **손익 계산**: `(현재가 - avg_cost) × quantity`
- **영속성**: 세션 내 유지 (앱 재시작 시 초기화 — 외부 저장소 미사용)
- **입력**: 선택 사항 (미입력 시 손익 계산 생략, 비중만 표시)
