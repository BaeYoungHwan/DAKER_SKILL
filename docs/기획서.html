1. 프로젝트 개요

1.1 한 줄 소개

Skills.md로 규칙을 정의하고, 바이브 코딩으로 구현한 실시간 금융 투자 분석 대시보드

1.2 목적

개인 투자자가 주식·ETF 데이터를 직관적으로 분석하고, 투자 의사결정을 지원하는 도구입니다.
별도의 API 키나 유료 서비스 없이 Yahoo Finance(yfinance) 를 통해 글로벌 및 한국 주식 실시간 데이터를 조회합니다.

1.3 핵심 차별점

항목 내용
API Key 불필요 yfinance 기반, 별도 인증 없이 즉시 실행
한국 주식 지원 한국어 종목명(삼성전자, 카카오 등) 검색 → 자동 티커 변환
혼합 통화 처리 KRW/USD 혼합 포트폴리오 실시간 환율 자동 적용
Skills 기반 자동화 분석·시각화·인사이트 규칙을 문서로 정의 → AI가 코드로 구현
자동 리포트 생성 분석 결과를 .txt 리포트로 다운로드

2. 바이브코딩 방법론

2.1 Skills.md → AI → Code 파이프라인

이 프로젝트는 "규칙을 먼저 문서화하고, AI가 코드를 생성" 하는 바이브코딩 방식으로 개발되었습니다.

Skills/analysis.md      →  투자 지표 계산 규칙 정의
Skills/visualization.md →  차트 선택 기준 및 색상 시스템 정의
Skills/insight.md       →  인사이트 등급·형식·우선순위 정의
        │
        ▼  (Claude Code — 바이브 코딩)
        │
src/data/fetcher.py     →  yfinance 데이터 수집 모듈
src/analysis/indicators.py  →  기술적 지표 계산 모듈
src/viz/charts.py       →  Plotly 차트 컴포넌트 모듈
src/app.py              →  Streamlit UI 통합 앱

2.2 Skills 런타임 파싱

Skills 문서에 정의된 수치(RSI 기준 14일, 과매수 70, 샤프비율 양호 기준 1.0 등)는
src/skills/parser.py 가 런타임에 마크다운을 파싱하여 코드에 주입합니다.

Skills/analysis.md 수정
        │ (앱 재시작 즉시 반영)
        ▼
skills/parser.py → load_analysis_config() → lru_cache
        │
        ▼
indicators.py · app.py → 동적 임계치 적용

→ 코드 수정 없이 분석 규칙 변경 가능 — Skills.md가 곧 설정 파일

2.3 Skills 문서 구성

Skills/analysis.md — 분석 규칙

  • 지원 데이터 유형 정의 (주가, ETF, 재무지표, 시장지수, 섹터)
  • 수익률 계산 공식 명세
  • RSI, 이동평균, 볼린저밴드, 변동성, 샤프비율 임계치 정의
  • 포트폴리오 분산도 평가 기준 (단일 종목 >30%, 섹터 >40% 경고)

Skills/visualization.md — 시각화 규칙

  • 데이터 유형별 차트 선택 기준 (캔들스틱/라인/히트맵/산점도)
  • TradingView 다크 테마 색상 토큰 시스템 (17개 색상 토큰 정의)
  • 차트 색상 상수 정의 (MA5/20/60/120/200별 색상)
  • 레이아웃 그리드 규칙 (2/3 : 1/3 분할 등)

Skills/insight.md — 인사이트 생성 규칙

  • 3단계 형식: 신호 → 해석 → 행동 제안
  • 등급별 임계치: High/Medium/Low
  • 인사이트 우선순위 규칙 (High 최상단, 종목당 최대 3개)
  • 리포트 5섹션 구성 흐름 정의

Skills/interaction.md — UI 상태 관리 규칙

  • Session State 초기화 키 정의 (selected_tickers, ticker_names, holdings 등)
  • 워치리스트 영속성: watchlist.json UTF-8 저장·로드 흐름
  • 검색 카운터 패턴: 종목 추가 후 검색창 자동 초기화
  • URL 파라미터 처리: ?tickers=005930.KS 앱 로드 시 자동 추가

모멘텀 배지 1주·1개월·3개월 수익률 (상승↑/하락↓ 색상)

MACD 차트 MACD·시그널·히스토그램

실적 캘린더 다음 실적 발표일 D-day + EPS 서프라이즈(예상 vs 실제) + 발표 14일 이내 변동성 경고

6. 안정성 및 데이터 품질

6.1 Rate Limit 핸들링

  • fetch_price(): 실패 시 최대 3회 자동 재시도 (지수 백오프)
  • fetch_multiple(): 종목 간 0.3초 딜레이로 API 과부하 방지

6.2 데이터 이상치 처리

  • 일별 수익률 ±15% 초과 시 이상치 플래그 (기준은 Skills/analysis.md에서 런타임 로드)
  • 이상치 경고: 종목당 최대 3건 + 접힘(collapsed) 표시로 UX 방해 최소화
  • 결측값: forward fill 처리 (Skills/analysis.md §4 기준)

6.3 오류 방어

  • 존재하지 않는 티커 입력 시 앱 크래시 없이 graceful 처리
  • 데이터 없는 재무항목은 UI에서 자동 미표시
  • 단일 종목 / 멀티 종목 모두 오류 없이 동작
